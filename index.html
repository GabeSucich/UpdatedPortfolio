Project Miles

"Project Miles" is the result of my first experience working on a coding team. Designed for young adolescents first starting to become conscious of the importance of nutrition and exercise, this application aims to engage young users to make healthier choices in their meals and activities. The project required responses from several different server-side API's to be integrated into a youth-friendly interface which gamified the process of making healthy choices. As the member of the team with the most experience with programming logic, I was delegated to code API response collection and integration on the back-end. However, due to newness of the team to coding, I ended up contributing heavily to the front-end as well. Though the process was chaotic, as a silver lining, I gained much experience with front- and back-end coding, and my contributions are rich throughout all of the source code!

Weather Dashboard

Here is a basic weather dashboard. It utilizes a weather-forecast API to display weather information to the user on the dashboard, and stores search queries in local storage for future reference. As an assignment early in my programming education, the task was an introduction to the use of API's and local storage. To stretch the confines of the assignment with my limited experience, I added case sensitivity to search queries, duplicate recognition, and front-end polishing for an attractive UI.

Code Quiz

This rudimentary "code quiz" was less a task to come up with engaging programming questions, and more an exercise in the newly developed skill of using JavaScript to dynamically update the DOM from local storage and from user input. Additionally, this application features a consistent timer which is responsive to user input as well. Though limited in its function by the fact that I developed it with only two weeks of front-end coding experience, the code for this application displays an attempt to be creative and structured with (retrospectively VERY) limited knowledge. Features like the modular-function flow of the gameplay and randomization of answer-choice ordering reflect my early attempts stretch current capabilities and model the structure of my simpler projects after more complex projects to come.
